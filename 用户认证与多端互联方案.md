## 一、项目背景

### 1.1 当前状态
- **项目名**: pg-tracker-v2
- **类型**: Electron 桌面应用（保研信息管理工具）
- **技术栈**: React 18 + TypeScript + SQLite + Prisma + Zustand + Tailwind CSS
- **现状**: 纯本地应用，无用户认证，无云端同步

### 1.2 目标
- 实现用户认证系统（邮箱登录）
- 实现数据云端同步
- 开发手机 App（React Native）
- 实现手机与电脑数据互联

### 1.3 用户选择
- ✅ 仅邮箱登录
- ✅ 云服务器部署

```prisma
  interviews        UserInterview[]
```

### 4.2 设备表


### 4.3 同步记录表

```prisma
model SyncRecord {
  id              String    @id @default(cuid())
  userId          String
  entityType      String    // INSTITUTION, ADVISOR, TASK, ASSET, INTERVIEW, EMAIL_TEMPLATE
  entityId        String    // 业务实体 ID
  operation       String    // CREATE, UPDATE, DELETE
  version         Int       // 操作版本号
  timestamp       DateTime  @default(now())
  checksum        String?   // 数据校验和

  @@unique([userId, entityType, entityId, version])
  @@index([userId, timestamp])
  @@map("sync_records")
}
```

### 4.4 业务数据表（以 Institution 为例）

```prisma
model UserInstitution {
  id              String    @id @default(uuid())
  userId          String

  // 业务字段
  name            String
  department      String
  tier            String    // REACH, MATCH, SAFETY (冲/稳/保)
  degreeType      String    // MASTER, PROFESSIONAL, PHD
  campDeadline    DateTime?
  pushDeadline    DateTime?
  expectedQuota   Int?
  policyTags      String    // JSON 格式存储标签

  // 同步字段
  localId         String?   // 本地 SQLite 的原始 ID（用于首次同步映射）
  version         Int       @default(0)
  deletedAt       DateTime? // 软删除
  lastModifiedAt  DateTime  @default(now())
  modifiedBy      String    // 设备 ID

  // 关联
  advisors        UserAdvisor[]
  tasks           UserTask[]

  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  @@unique([userId, id])
  @@index([userId, tier])
  @@map("user_institutions")

```

> 其他业务表（Advisor、Task、Asset、Interview、EmailTemplate）结构类似，都包含 userId 和同步字段。

### 6.2 Token 机制

| Token 类型 | 有效期 | 用途 | 存储位置 |
|-----------|-------|------|---------|
| accessToken | 15 分钟 | API 认证 | 内存 / AsyncStorage |
| refreshToken | 7 天 | 刷新 accessToken | 安全存储 (Keychain/Keystore) |

**Token 刷新流程**:
1. accessToken 过期前 5 分钟，客户端自动刷新
2. 使用 refreshToken 换取新的 accessToken + refreshToken
3. 旧的 refreshToken 加入 Redis 黑名单
4. 每个 refreshToken 只能使用一次

### 7.3 冲突解决策略

| 冲突类型 | 解决策略 | 说明 |
|---------|---------|------|
| 不同字段修改 | 自动合并 | A 修改 name，B 修改 department → 合并 |
| 相同字段修改 | 后写入者胜出 | 最后修改时间戳决定 |
| 删除 vs 更新 | 删除优先 | 一方删除，一方更新 → 删除 |
| 批量冲突 | 提示用户 | 展示差异，用户选择 |

**冲突检测代码示例**:
```typescript
function detectConflict(
  serverVersion: SyncableEntity,
  clientVersion: SyncableEntity,
  baseVersion: SyncableEntity
): string[] {
  const conflicts: string[] = []

  for (const key of Object.keys(serverVersion)) {
    const serverChanged = serverVersion[key] !== baseVersion[key]
    const clientChanged = clientVersion[key] !== baseVersion[key]
    const bothChanged = serverVersion[key] !== clientVersion[key]

    if (serverChanged && clientChanged && bothChanged) {
      conflicts.push(key)
    }
  }

  return conflicts
}
```

### 7.4 离线模式支持

```typescript
class SyncManager {
  private isOnline: boolean = true
  private pendingChanges: SyncQueueItem[] = []

  // 本地变更处理
  async handleChange(entity: SyncableEntity, operation: Operation) {
    // 1. 写入本地数据库（乐观更新）
    await this.localDb.save(entity)

    // 2. 加入同步队列
    await this.syncQueue.add({
      entityType: entity.type,
      entityId: entity.id,
      operation,
      payload: JSON.stringify(entity),
      timestamp: new Date(),
      status: 'PENDING'
    })

    // 3. 如果在线，立即同步
    if (this.isOnline) {
      this.sync()
    }
  }

  // 网络状态变化
  async onNetworkStatusChange(online: boolean) {
    this.isOnline = online
    if (online) {
      await this.processPendingChanges()
    }
  }
}
```

| 安全措施 | 实现方式 |

### 8.2 API 安全

```typescript
// Rate Limiting
@Injectable()
export class RateLimiterMiddleware implements NestMiddleware {
  async use(req: Request, res: Response, next: Function) {
    const key = `rate-limit:${req.ip}:${req.path}`
    const requests = await this.redis.incr(key)

    if (requests === 1) {
      await this.redis.expire(key, 60) // 1 分钟窗口
    }

    if (requests > 100) { // 每分钟 100 次限制
      throw new TooManyRequestsException()
    }

    next()
  }
}
```

### 8.3 文件上传安全

```typescript
@Injectable()
export class FileUploadService {
  private readonly allowedMimeTypes = [
    'application/pdf',
    'application/msword',
    'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
    'image/jpeg',
    'image/png'
  ]

  private readonly maxFileSize = 10 * 1024 * 1024 // 10MB

  async uploadFile(file: Express.Multer.File, userId: string) {
    // 验证 MIME 类型
    if (!this.allowedMimeTypes.includes(file.mimetype)) {
      throw new BadRequestException('不支持的文件类型')
    }

    // 验证文件大小
    if (file.size > this.maxFileSize) {
      throw new BadRequestException('文件大小超过限制')
    }

    // 生成安全文件名
    const safeName = `${uuid()}${path.extname(file.originalname)}`
    const key = `users/${userId}/${safeName}`

    // 上传到对象存储
    await this.ossClient.put(key, file.buffer)

    return { key, url: this.getSignedUrl(key) }
  }
}
```

## 十三、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 同步冲突复杂 | 数据不一致 | 完善冲突检测，提供手动解决 UI |
| 离线数据丢失 | 用户数据丢失 | 本地定期备份，云端版本历史 |
| Token 泄露 | 账号安全 | 短期 Token + 黑名单 + 设备管理 |
| 服务器宕机 | 服务不可用 | 负载均衡 + 自动重启 + 监控告警 |
| 手机 App 审核不通过 | 上线延迟 | 提前了解审核规则，准备材料 |
